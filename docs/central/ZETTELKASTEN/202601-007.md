## Fakt

Proces budowy bazy transkrypcji z YouTube z profilowaniem ekspertów:

### Pipeline

```
1. INPUT: URL wideo / kanał / playlist
           ↓
2. FETCH: yt-dlp / YouTube API → metadata + transkrypcja
           ↓
3. PROCESS: Claude → wyciągnięcie kluczowych wniosków
           ↓
4. PROFILE: Identyfikacja ekspertów/rozmówców
           ↓
5. STORE: ZETTELKASTEN + EXPERTS_DB
           ↓
6. INDEX: KNOWLEDGE_GRAPH aktualizacja
```

### Struktura transkrypcji

```
docs/central/TRANSCRIPTS/
├── YYYYMMDD-channel-title.md
├── EXPERTS/
│   ├── john-doe.md (profil eksperta)
│   └── jane-smith.md
└── _INDEX.md
```

### Format transkrypcji

```markdown
# [Tytuł wideo]

**source:** https://youtube.com/watch?v=XXX
**channel:** [Nazwa kanału]
**date:** YYYY-MM-DD
**duration:** HH:MM:SS
**confidence:** 0.65

## Rozmówcy
- **John Doe** (→ EXPERTS/john-doe.md) — CEO, AI researcher
- **Jane Smith** — Host

## Kluczowe wnioski
1. [Wniosek 1] (timestamp: MM:SS)
2. [Wniosek 2]

## Cytaty
> "Cytat 1" — John Doe, 05:23

## Pełna transkrypcja (collapsed)
<details>
<summary>Rozwiń transkrypcję</summary>
[Full transcript...]
</details>
```

### Format profilu eksperta

```markdown
# [Imię Nazwisko]

**role:** [CEO / Researcher / Developer...]
**expertise:** [AI, ML, Business...]
**confidence:** 0.70
**sources:** [lista transkrypcji gdzie występuje]

## Bio
[Krótka biografia]

## Kluczowe poglądy
- [Pogląd 1]
- [Pogląd 2]

## Cytaty
- "[Cytat]" — źródło

## Powiązania
- → [inne osoby, firmy, projekty]
```

## Kontekst

Dla Fazy 3 (RAG) transkrypcje YT to cenne źródło:
- Wywiady z ekspertami (unikalna wiedza)
- Tutoriale techniczne
- Konferencje

Confidence niższe (0.65) bo:
- Auto-transkrypcja może mieć błędy
- Opinie mogą być subiektywne
- Nieaktualne informacje
