## Fakt

Jarvis może czerpać wiedzę z wielu źródeł. Architektura dwukierunkowa:

### 1. Źródła zewnętrzne (Jarvis podłącza się)
- Dokumentacja API (Anthropic, n8n, Google...)
- GitHub repositories
- Stack Overflow
- Blogi techniczne

### 2. Źródła budowane (Jarvis tworzy)
- ZETTELKASTEN (atomowe fakty)
- Transkrypcje YT z profilami ekspertów
- Digesty z emaili/kalendarza
- ADR (decyzje architektoniczne)

### Confidence scoring wg źródła

| Źródło | Base Confidence | Uzasadnienie |
|--------|-----------------|--------------|
| Oficjalna docs | 0.95 | Weryfikowane przez vendor |
| GitHub code | 0.90 | Można sprawdzić |
| Tech blogs | 0.75 | Opinie, mogą być nieaktualne |
| YT transcripts | 0.65 | Mogą być błędy w transkrypcji |
| User discussion | 0.60 | Hipotezy, niezweryfikowane |

## Kontekst

W Fazie 3 (MÓZG) Jarvis powinien:
1. Aktywnie szukać źródeł wiedzy (web search, fetch docs)
2. Budować własne źródła (zettelkasten, transkrypcje)
3. W odpowiednich momentach podłączać się do obu typów
