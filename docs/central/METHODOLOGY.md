## Design Thinking — 5 etapów

| Etap | Opis | Pytania |
|------|------|---------|
| **1. Empathize** | Zrozum użytkownika | Kto ma problem? Jak się czuje? |
| **2. Define** | Zdefiniuj problem | Jaki jest prawdziwy problem? |
| **3. Ideate** | Generuj pomysły | Jak można to rozwiązać? (ilość!) |
| **4. Prototype** | Zbuduj prototyp | Co możemy szybko zbudować? |
| **5. Test** | Testuj z użytkownikami | Co działa? Co nie? |

## Techniki burzy mózgów

### 1. Klasyczna burza mózgów (Brainstorming)
- Ilość przed jakością
- Żadnej krytyki
- Budowanie na pomysłach innych
- Szalone pomysły mile widziane

### 2. Odwrócona burza mózgów
Zamiast "Jak rozwiązać problem?" pytamy:
- "Jak moglibyśmy POGORSZYĆ sytuację?"
- Potem odwracamy odpowiedzi

### 3. SCAMPER
| Litera | Pytanie |
|--------|---------|
| **S** - Substitute | Co można zastąpić? |
| **C** - Combine | Co można połączyć? |
| **A** - Adapt | Co można zaadaptować? |
| **M** - Modify | Co można zmodyfikować? |
| **P** - Put to other use | Do czego innego można użyć? |
| **E** - Eliminate | Co można usunąć? |
| **R** - Reverse | Co można odwrócić? |

### 4. Six Thinking Hats (de Bono)
| Kolor | Perspektywa |
|-------|-------------|
| Biały | Fakty, dane |
| Czerwony | Emocje, intuicja |
| Czarny | Krytyka, ryzyka |
| Żółty | Optymizm, korzyści |
| Zielony | Kreatywność, alternatywy |
| Niebieski | Proces, organizacja |

## Łączenie kropek

Steve Jobs w słynnym przemówieniu na Stanford (2005):
> "You can't connect the dots looking forward; you can only connect them looking backwards."

### Techniki łączenia:
1. **Forced connections** — weź dwie losowe rzeczy, znajdź połączenie
2. **Analogie** — jak ten problem rozwiązano w innej branży?
3. **Transfer technologii** — co z dziedziny X można zastosować w Y?

---

## Pytania otwierające

Używaj przy rozkminkach:

- "A co gdyby...?"
- "Co by było, gdyby nie było żadnych ograniczeń?"
- "Jak rozwiązałby to 5-latek?"
- "Jak rozwiązałby to Elon Musk / Steve Jobs / [ktoś]?"
- "Co by się stało, gdybyśmy zrobili dokładnie odwrotnie?"
- "Jaki jest ukryty problem za tym problemem?"
- "Dla kogo to jest naprawdę problem?"
- "Jak to będzie wyglądać za 10 lat?"

## Prototypowanie

### Zasady:
- **Szybko i tanio** — nie buduj finalnego produktu
- **Fail fast** — im szybciej się dowiesz że nie działa, tym lepiej
- **Show, don't tell** — pokaż zamiast opowiadać

### Formy prototypów:
- Szkic na kartce
- Mockup w Figma
- Spreadsheet symulujący logikę
- Prosty skrypt
- Workflow n8n (draft)
